# 踩坑：截断导致的问题

> 🚧 Work in Progress

截断不是万能药，处理不当会引入新的问题。

## 常见陷阱

### 1. 幻觉（Hallucination from Missing Context）

截断丢掉了关键上下文，模型不知道之前做过什么，开始"编"。

**典型场景**: 多步任务中间截断了前置步骤的结果，模型以为还没做，重复执行或跳过。

**解法**: 压缩后恢复关键上下文（Claude Code 的"恢复最近5文件"机制）。

### 2. 注意力稀释（Attention Dilution）

即使没截断，上下文太长也会导致模型注意力分散——著名的 lost-in-the-middle 问题。

**典型场景**: RAG 塞了 20 条结果，模型只看了开头和结尾。

**解法**: RAG 结果少而精（5-8条），而非多而全。

### 3. 信息冲突（Context Clash）

截断后的摘要与后续新增信息矛盾。

**典型场景**: 摘要说"用户偏好A"，但最新对话中用户改口偏好B，摘要没更新。

**解法**: 摘要要标注时效性，新信息覆盖旧摘要。

### 4. 工具输出丢失

工具返回大段输出被截断后，模型无法引用工具结果做下一步决策。

**解法**: 大输出落盘，上下文只保留摘要+路径，需要时再读取。
