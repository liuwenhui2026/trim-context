# 成本分析：截断 ≠ 省钱

> 🚧 Work in Progress

截断上下文省了输入 Token，但可能增加总成本——因为信息丢失导致任务失败需要重试，或者摘要本身消耗额外 Token。

## 成本构成

| 策略 | 输入 Token | 输出 Token | 额外 API 调用 | 任务成功率 |
|:-----|:----------|:----------|:-------------|:----------|
| 不截断 | 高 | 正常 | 0 | 最高（但可能超限） |
| Sliding Window | 低 | 正常 | 0 | 看场景 |
| Summary Compress | 中 | 中 | +1（摘要调用） | 中高 |
| Isolate Fork | 主低+子高 | 正常 | +N（子Agent） | 高 |

## 隐藏成本

- **重试成本**: 截断导致信息丢失 → 回答质量下降 → 用户追问 → 更多轮次 → 总成本反而更高
- **摘要成本**: 生成摘要本身需要一次 LLM 调用
- **子Agent成本**: Isolate Fork 的子 Agent 调用是额外的
