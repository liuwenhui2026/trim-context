# Claude Code: 4 级渐进压缩

> 🚧 Work in Progress

Claude Code 是目前上下文管理做得最精细的生产系统。当上下文逼近窗口上限时，不是一刀切，而是分 4 级逐步处理：

1. **裁剪（Trim）** — 截断历史消息中的大块内容（旧的工具输出）
2. **去重（Dedup）** — 几乎零成本地去除重复内容
3. **折叠（Collapse）** — 把不活跃的对话段落折叠，但不修改原始内容
4. **摘要（Autocompact）** — 启动子 Agent 对整个对话做摘要

每一级都可能释放足够空间，后续级别无需执行。压缩后自动恢复最近编辑的 5 个文件内容，防止模型"失忆"。

## 源码关键路径

- `src/services/compact/` — 压缩流水线实现
- `src/context.ts` — 上下文构建与 memoize 缓存

## 缓存与截断的张力

Claude Code 依赖前缀缓存（KV Cache）降低延迟和成本。但缓存要求前缀字节级完全一致——随便调一下工具顺序就会导致缓存全部失效。

所以截断不是简单的"删掉老的"，而是"删掉不影响缓存的部分"。这是 Claude Code 上下文工程最精妙的地方。
