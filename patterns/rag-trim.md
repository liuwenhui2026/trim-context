# RAG Trim: 检索结果的截断策略

> 🚧 Work in Progress

RAG 和上下文工程的交汇点——检索到了很多结果，但上下文放不下，怎么截？

## 问题

RAG 检索到 20 条结果，上下文窗口只放得下 5 条。截哪些？截完怎么排？跟对话历史怎么共存？

## 策略

- 按 Reranker 分数截断：只保留 Top-K
- 按相关性衰减：越相关的越长，不太相关的只保留摘要
- 动态预算分配：根据对话历史长度动态调整 RAG 结果的 Token 预算
- 分层加载：先给摘要，需要时再加载原文

## 与 Wolf-Crying 问题

RAG 结果即使相关，塞太多也会导致"注意力稀释"——模型被海量参考文本分散注意力，回答质量反而下降。截断不止是空间问题，更是注意力问题。
